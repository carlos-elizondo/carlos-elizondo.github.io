Creating a modern CRUD Watchlist Web App

In this project I designed a CRUD Web App watchlist with a MongoDB database.

Description

For this project I used various JavaScript libraries with Node.js:

  • I used Axios to make HTTP requests from node.js which supports the Promise API that is native to JS ES6.
  • I used EJS (Embedded JavaScript Templating) which is one of the most popular template engines for JavaScript. Which can be used to embed JavaScript code in a template language that is then used to generate HTML.
  • I used Express which is a minimal and flexible Node.js web application framework for building RESTful APIs that provide a robust set of features for web and mobile applications.
  • I used Mongoose, an Object Data Modeling (ODM) library for MongoDB to manage my database.
  • I used MongoDB which is an open source NoSQL database management program. MongoDB stores data in flexible, JSON-like documents.

Note: GitHub Pages only host static pages, this is why for the implementation you are about to see you will not be able to add, edit, or delete an entry because it is not connected to a database.

Create, Read, Update, and Delete (CRUD) are the four basic functions to be considered when building APIs and that the models should be able to do.

In a REST environment these basic functions corresponds to the HTTP methods POST, GET, PUT, and DELETE.

On the homepage, the watchlist connects to the database and displays (reads) the movies or TV shows you have added to it. Here we can see that our database contains two entries: “Mission: Impossible - Dead Reckoning Part One” and Secret Invasion. Each entry has an ID, Poster Image, Title, Release Date, and Content Type associated with it.

If you want to add (create) a new entry there is a button on the left-top corner of the interface that says “New Entry” once you click that button you will be able to add a new entry by providing the 4 pieces of information each entry has to have. Once you click the save button it would send this information to the MongoDB database.

You can also edit (update) an entry, for example, “Secret Invasion” which is not a movie but a TV show to do this you would click on the “pencil” button corresponding to that entry and this would grab the information associated with that entry so that you could easily edit any field. Now in this example, we could change the content type to “TV Show” and click save and if it was connected to the database it would edit this entry and update it. Because it is not connected to the database the content type for “Secret Invasion” remains a movie.

If you wanted to delete any entry you would only need to click on the “delete” button below the edit button and this would delete the entry from the database.
